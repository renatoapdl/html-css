Getting started

Getting started

Thank you for purchasing the Free version of Greater Love Church website template. This template can be used to create any church website whether it be a non-denominational or denomination.

This is just a template coded with HTML, CSS, and JavaScript therefore additional scripting still needed to make it full dynamic and functioning website.

Text editor

To edit this files for this template, you will need a text editor. There are many free text editors out there. Here some of the most popular ones:

  • Visual Studio Code
  • Atom.io
  • Brackets
  • Sublime Text
  • Notepad++

The CSS for this template is written in SASS (SCSS) and compiled into regular CSS. To compiled SCSS files you will need a SASS compiler. A popular free option is Prepros.

Domain & hosting

To put your website online to the world to see, you will need a hosting account and a domain name.

A hosting account will allow your website files to be save on another computer that will always start on, therefore your website will be online.

A domain name is your website address or name. For example, Google’s domain name is “google.com”.

Here is a list of popular companies that offer hosting and domain registration:

  • GoDaddy
  • HostGator
  • TMDHosting
Markup

Html

All the HTML files that make up the website have an .html extension. The homepage is called index.html

To change language for your document, follow below... “lang” is the language of your website, by default it is set to “en” for English.

              
<!DOCTYPE html>
<html lang="en"> 
  <!-- The "lang" is the language of your website -->
  ...
</html>

To change the page title:

              
<!DOCTYPE html>
<html lang="en">
  <head>  
    ...  
    <!-- Your page title -->
    <title>Your page title here</title>  
  
  </head>
  ...
</html>

A favicon is the small image at the top left corner of a browser tab. To change the favicon:

              
<!DOCTYPE html>
<html lang="en">
  <head>  
    ...  
    <!-- Place Custom Favicon here (.png, .jpg, or .svg) -->
    <link rel="shortcut icon" href="path/to/favicon" type="image/png">
  
  </head>
  ...
</html>

This template comes with page loader that load while your website contents are being loading. To remove it, simply do this:

              
<!DOCTYPE html>
<html lang="en">
  <head>  
    ...
  </head>
  <body>
  <!-- Remove the following code below -->
    <div class="page-loader">
      <div class="loader"><img src="images/logo-sm.svg" alt=""></div>
    </div>
    ...
  </body>
</html>

Header (navbar)

The header or navbar that comes with this template is dynamic and flexible. It allow you to add classes or change the logo on scroll. You can change its background color, text color, and etc. The SCSS file that governs the header is located at scss/layout/_headers.scss

These are the predefined classes you can use on the header:

  • transparent - will make the navbar transparent
  • white-bg - will give the navbar a white background color with drop shadow
  • fixed - will make the navbar stick to the top and follows as you scroll
  • light-text - will make the navbar menu item lighter color
  • dark-text - will make the navbar menu item darker color

By default the header is transparent, so you can omit that class if you want it to be transparent. The header needs the header class otherwise it won’t work.

<header class="header">...</header>

To add classes to the header or change the logo on scroll:

<header class="header" data-onscroll-classes="class1 class2" data-onscroll-logo="path/to/your/logo">...</header>

Note: If you choose not to add any classes you can delete the whole data-onscroll-classes or if you choose not to change the logo, you can delete the whole data-onscroll-logo.

To change the default logo:

              
<!DOCTYPE html>
<html lang="en">
  <head>  
    ...
  </head>
  <body>
    ...
    <header class="header">    
      <div class="container">    
        <nav class="header__nav bottom-nav">
    
          <div class="header__logo brand--logo">
            <!-- Change your logo here -->
            <a href="index.html"><img src="path/to/your/logo" alt="Brand logo"></a>
          </div>
          ...
        </nav>
      </div>
    </header>
  </body>
</html>

To add new page to the menu or update an existing page:

              
<!DOCTYPE html>
<html lang="en">
  <head>  
    ...
  </head>
  <body>
    ...
    <header class="header">    
      <div class="container">    
        <nav class="header__nav bottom-nav">    
          ...
          <ul class="header__navitems show-on-lg" id="mobile-menu">
            <!-- Add new or update existing menu items -->
            <li class="header__list"><a href="page-url">New page name</a></li><!-- .header__list ends -->
            ...
          </ul>
        </nav>
      </div>
    </header>
  </body>
</html>

To add a dropdown menu to a menu item:

              
<header class="header">      
  ...
  <!-- Add a submenu. You need to add "dropdown-link" class to the anchor tag that will be click  -->
  <li class="header__list">
    <a href="" class="dropdown-link">Home</a>
    <div class="header__submenu"><!-- Submenu -->
      <!-- You can place anything here -->
      ...        
    </div><!-- .header__submenu ends -->
  </li><!-- .header__list ends -->
  ...
</header>

Important: All this JavaScript (plugins and custom) files are place before the closing of the body tag to ensure faster page load.

Styling

Css & scss

The styling for this template was written in SASS (SCSS), compiled into regular CSS.

The compiled CSS can be found here dist/css/main.css – it is not commented

The compiled css is already linked inside the HTML document inside the head tags. To add your custom css file:

              
<!DOCTYPE html>
<html lang="en">
  <head>
    ...            
    <!-- Main CSS -->
    <link href="dist/css/main.css" rel="stylesheet">
    <!-- Your custom CSS -->
    <link href="path/to/your/css" rel="stylesheet">
  ...
</html>

The scss folders and files are located inside the scss folder – the scss files are commented

The main.scss is use to import the Google font, the icon fonts, and all the other scss files (partials) to be compiled into the main.css file

          
            // main.scss file 
// Importing Google font - Roboto Condensed & Lobster Two || You can change the Google fonts here
@import url("https://fonts.googleapis.com/css?family=Lobster+Two|Roboto+Condensed:400,700&display=swap");
// Importing Remix Icon ||We are using Remix Icons - you can change here
@import url("https://cdn.jsdelivr.net/npm/remixicon@2.3.0/fonts/remixicon.css");
...

To change the colors and font sizes, go to the _variable.scss file located inside scss/abstracts/

The "pages" folder inside the scss folder holds folders for each page on the website. And its page folder holds a scss file for the different section on that page. For example, if you want to edit the “Next event” section on the first homepage, simply locate the _next-event.scss file inside scss/pages/home/

JavaScript

JavaScript

Some of the plugins used in this template requires JQuery therefore you will need the JQuery library. By default, it is included. It should be include first before including other JavaScript files. The main.min.js is the minified version and it is loacted inside dist/js/.

To edit the non-minified, locate js/main.js. Prepros was used to compile and minified it. If you choose not to use the minified, simply update your script tag to point to the un-minified.

Note: The "main" script file is needed for certain components to work such as the header. If you choose to create your own JS file and remove the original "main" script file, copy the "main" file and paste in your custom JS file.

              
<!DOCTYPE html>
<html lang="en">  
  ...
  <body>
    ...

    <!-- JQUERY -->
    <script src="https://code.jquery.com/jquery-3.3.1.min.js"></script>
    <!-- loading in JQuery locally if CDN failed -->
    <script>window.jQuery || document.write('<script src="js/jquery.min.js">\x3C/script>')</script>
    ...
    <!-- Main (custom) script - minified -->
    <script src="dist/js/main.min.js"></script>
    <!-- Main (custom) script - unminified -->
    <script src="js/main.js"></script>
    <!-- Include your own script file here if you choose to -->
    <script src="path/to/your/js/file"></script>
  </body>
</html>
Grid

Grid layout

This template doesn’t use Bootstrap framework but rather use simple and flexible custom framework. It is a mobile first which includes a responsive 12 columns grid. If you use Bootstrap, then it will be simple to understand this framework.

These are the breakpoints and you can adjust them to your liking inside scss/abstracts/_variables.scss:

              
// GRID LAYOUT ======================== //      
...

// Breakpoints
$sm: 0em; // mobile phone - portrait
$md: 34.5625em; // 553px / 16px (medium devices)
$lg: 56.3125em; // 901px / 16px (large devices)
$xl: 87.5625em; // 1401px / 16px (extra large devices)

It has a fixed container and these are the width on different devices:

              
// GRID LAYOUT ======================== //      
...

// Breakpoint - max-width for container
$width-container-sm: 90%;
$width-container-md: 85%;
$width-container-lg: 123.3rem; // 1233px = 1233px (desired max-width) / 10px (base font size) ~ 123.3rem
$width-container-xl: 130rem; // 1560px = 1560px (desired max-width) / 12px (base font size) ~ 130rem

Rows and columns

The row is displayed as a flexbox, therefore anything immediately inside of it will become a flex item. 

              
<div class="row">...</div>

The columns need to be inside a row. To declare a column: flex-{device-width}-{number-of-col}

              
<div class="row">
  <div class="flex-sm-2 flex-md-3 flex-lg-4 flex-xl-5">
    ...
  </div>
</div>
Components

Components

This template comes with handful of component elements such as buttons, cards, pagination, and etc. In this section, we will go over few of them

Button

To create a button, you just need to add the button class

<a href="" class="button">Click me</a>

The button also comes with block level option. If you wish to have the button to be 100% of its container width, look at the code below

<a href="" class="button button-block-sm button-inline-md">Click me</a><!-- Will display as block on smaller devices but display as inline (regular) on medium and upper devices. --><!-- Options includes button-block-{sm | md | lg | xl} and button-inline-{sm | md | lg | xl}. -->

Forms (inputs)

It is very easy to create an input field, simply add the form__input class to the input. See the code below

<input type="text" class="form__input" placeholder="...">

It is very easy to create a textarea field, simply add the form__input form__textarea classes to the textarea. See the code below

<textarea name="..." class="form__input form__textarea" placeholder="..."></textarea>

Creating a custom radio button, see the code below

              
<div class="form__group">
  <input type="radio" name="custom-radio" class="form__radio" id="custom-radio">
  <label for="custom-radio" class="form__label form__label--radio">Custom radio btn</label>
</div><!-- .form__group ends -->

Creating a custom checkbox, see the code below

              
<div class="form__group">

  <label class="custom-label form__checkbox">
    Your label goes here
    <input type="checkbox" checked>
    <span class="checkmark"></span>
  </label>

</div><!-- .form__group ends -->

Lightbox using LightboxJS

To create the lightbox, first wrap your image inside an anchor tag:

              
<a href="path/to/your/larger/image" data-lightbox="church-images" data-title="Your image caption">Your image (smaller version)</a>
<!-- "data-lightbox" is used if you have a group of related images that you want to combine and "data-title" is used if you want to add caption to your image -->

Animate On Scroll using AOS

This is use for animation. You can animate elements on your website as you scroll through the page. The code below shows how to add a simple "fade-up" animation:

              
<div data-aos="fade-up"></div>
<!-- you set the animation using the "data-aos" attribute -->

You can change the animation behaviour by using data-aos-* attributes on each elements:

              
<div data-aos="fade-up" data-aos-offset="400" data-aos-delay="40" data-aos-duration="1500" data-aos-easing="ease-in-out">
</div>
